1. 计数器

众所周知，计算器可以用Ans进行一元一阶递推数列的计算。但是当要计算的次数(递推的次数)非常大，仅使用Ans计算难免会数错，引入一个计数器是个好方法。计数器可以随着递推式的计算同时输出计算的次数，代价是更多的点击次数。

2. 如何使用一个计数器

2.1 提示

  1. 为了让编辑语句和重复计算的界面更方便看，可以打开线性输入输出和小字体（并非必要）。

    • 按键SHIFT 设置 1 3 即可设置为线性输入/输出，而SHIFT 设置 1 1 是平时常用的数学输入/输出。
    • 按键SHIFT 设置 下 下 4 2 即可设置为小字体，而SHIFT 设置 下 下 4 1 是平时常用的普通字体。

  2. 本节涉及到的按键及位置如下：

    • STO，它在功能键的左下角。
    • CALC，它在功能键第一排左二。
    • :，它在功能键第一排右二，需要先按ALPHA。
    • x²，即x2，它在功能键第二排左三。
    • ┚，即分数线，它在功能键第二排左一。
    • )，它在功能键最后一排右三。
    • x ，它在功能键右上角。
    • y ，它在功能键最后一排右二，需要先按ALPHA。
    • SHIFT、ALPHA，它们在左上角。
    • 以及下方数字键区域。

2.2 原理

STO键提供了把计算结果赋值给某一个变量的功能。

例如：依次按1 + 1 STO x，屏幕中显示：

xxxxxxxxxx
1+1→x

此时按下=计算，结果2将被赋值给x。

注意赋值给y或其他变量符号时，不要按ALPHA，直接按变量符号对应的按键。

同时，:能够分割语句，做到编辑多条计算语句后以此执行，整体称为一个语句块。

例如：依次按x × 2 STO x ALPHA : y + 1 STO y，屏幕中显示（线性输入下，数学输入略）：

xxxxxxxxxx
x×2→x:
y+1→y

重复按=，x的值将被乘二，y的值将被加一，重复。

当y的初始值等于0或1时(视情况而定)，y即代表了整个语句块计算的次数，或递推数列的下标，称为计数器。

2.3 示例

需要两条赋值语句，第一条为递推，第二条是计数器。

递推数列与赋值语句

利用赋值语句处理递推数列的思路是：把一次计算的结果代入下一次计算，即赋值符号两边同时有变量符号。

如x+x²/2023→x循环执行，不断地把x+x²/2023的值赋给新的x，对应了

an+1=an+(an)22023

再如A(1-x)→A，处于一个语句块中，每次执行这个语句块会更新x，而新的A用新的x计算，当A的初始值为1时，这个语句对应了

Tn=(1−a1)(1−a2)···(1−an)

因为这个式子的递推式可以写成Tn=Tn-1(1-an)

完整输入

完整输入如下：

(1. 递推式) x + x x² ┚ 2 0 2 3 STO x 左(让光标回到行末) ALPHA :(线性输入下此时自动换行)

(2. 计数器) ALPHA y + 1 STO y =(让语句块执行完) 左(让光标回到行末)

(3. 赋初始值) CALC 0 . 5 =(输入x的初始值0.5) 1 =(输入y的初始值1) =(赋值计算) 左(从错误界面回到语句)

屏幕显示如下（线性输入下，数学输入略）：

xxxxxxxxxx
x+x²┚2023→x:
y+1→y

然后就可以开始重复按=了，输出结果如下：

xxxxxxxxxx
0.5001235788
2
0.5002472188
3
…

直到按倒

xxxxxxxxxx
…
0.99……
2024
1.00……
2025

可以得出答案2025。

3. 解决递推数列问题的其他技巧

3.1 条件判断

计算器当然不能实现真正的条件判断，但是可以通过根号、对数等在结果符合条件后报错，实现及时自动刹车。

如上面的例子只需将第二行改成：

xxxxxxxxxx
y+1+0ln(1-x)→y

就会在x大于1时报错，只需按AC 上 上 就能看到上一个计数器的计数2024，得出答案2025，不用常常关注x的值看有没有按到，也不怕不小心按过了。

3.2 特殊的变量

  1. Ans，Ans永远将继承上一个运算结果，用于计算一元一阶递推数列很方便，在多条赋值语句中也有利用的空间，有时可以减少语句数量。
  2. M，M的特殊性在于它不仅像x、y等变量一样能够长期储存数据，它还拥有功能键M+和M-，它们的用途是分别将赋值语句M+x→M和M-x→M替换为xM+和xM-（x可以是任意表达式），区别则是使用M+和M-得到的计算结果是x的结果，而不是前两句赋值语句中M-x的结果，M的值会被更新但不会呈现。如果要检查M的值，直接输入单个M计算或使用SHIFT 调用查看。有时通过M+或M-和Ans的搭配使用可以减少语句数量。

3.3 Pol和Rec

这两个键是直角坐标和极坐标的互相转换，搭配使用可以做到类似一行同时执行两个赋值语句的效果（多条赋值语句是依次执行），详见https://www.bilibili.com/video/BV1aT411u7tJ

3.4 多元？多阶？

多条赋值语句当然不限于一元一阶递推，例如斐波那契数列是二阶递推 an+2=an+1+an （线性输入下，数学输入略）：

xxxxxxxxxx
x+y→x:
x-y→y

x的输出即为斐波那契数列。

可以使用的变量有9个，提供了极大的利用空间，大部分递推数列都可以用这种方法计算。

4. 提升计算器运算速度的技巧

4.1 减少语句数量

多条赋值语句中，每多一条语句，就需要在每个“循环”中多按一次运算键。当需要的计算次数非常大时，赋值语句的数量太多是非常致命的。

例如3.4中提到的斐波那契数列，现在有两个不同的方法。

方法A：

x
x+y→A
x→y
A→x

方法B：

xxxxxxxxxx
Rec(Pol(x+y,x),y)

方法A和方法B都能做到算出斐波那契数列，但是方法A需要每按3次才能计算出一次结果，方法B只要按1次就能出一次结果，假如都要算到第1000项，方法A需要按大约3000下，而方法B只需约1000下。

减少语句数量的方法有：变量的重复使用(如3.4中斐波那契数列的示例)、Ans和M的合理使用、Rec和Pol的使用(如方法B)等。

4.2 尽量使用基本运算

也不在这多提“时间复杂度”之类的东西，只需要知道计算器计算不同的运算类型所花费的时间是不同的。

加法和减法是最基本的运算，只需基本的二进制计算，忽略运算时间。

乘法和除法运算时间是相似的，数字越大，运算时间越长，但也可以忽略运算时间。

根号的本质是牛顿法找平方根，数字越大，运算时间越长，运算时间比乘除法长的多，特别是在SOLVE中，尽量两边平方后再计算，会显著提高运算时间。

对数用到了泰勒级数，运算时间更是远大于乘除法，SOLVE时尽量化为乘幂计算。
